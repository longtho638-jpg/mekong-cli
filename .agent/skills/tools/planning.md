# üìã Planning Skill

Strategic project planning and task breakdown.

## When to Use

- Starting new projects
- Complex feature development
- Architecture decisions
- Sprint planning

## Planning Framework

1. **Goal Definition** - What success looks like
2. **Requirements** - Must-have vs nice-to-have
3. **Architecture** - Technical approach
4. **Phases** - Sequential steps
5. **Risks** - Blockers and mitigations

## Output Template

```markdown
# [Project Name] Plan

## Goal
One sentence describing success.

## Requirements
- [ ] Core requirement 1
- [ ] Core requirement 2

## Architecture
Tech stack and design decisions.

## Phases
1. Phase 1: Foundation
2. Phase 2: Core Features
3. Phase 3: Polish

## Risks
| Risk | Mitigation |
|------|------------|
| X | Y |
```

## Example Prompts

```
"Use planning to create a CRM migration plan"
"Use planning to design API architecture"
"Use planning to scope MVP features"
```
