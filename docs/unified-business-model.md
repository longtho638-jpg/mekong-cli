## 2. MỐI LIÊN HỆ Mekong ↔ AgencyOS


## 3. MÔ HÌNH KINH DOANH OPEN-SOURCE RaaS

### Revenue-as-a-Service (RaaS) là gì?

**Definition:** Monetize AI agent work by charging credits per task executed.

**Unit Economics:**
```
User buys 100 credits = $99/month
Task complexity:
- Simple (1 credit): Single file edit, git operation
- Standard (3 credits): Multi-step feature
- Complex (5 credits): Full-stack with tests

AgencyOS margin: ~75% after contributor share
Contributor pool: ~20% of revenue
Platform ops: ~5% (hosting, support)
```

### Contributor Revenue Share

| Contribution | Share % | Example | Tracking |
|--------------|---------|---------|----------|
| **Core Engine (PEV)** | 5% | Improve planner.py | Git blame on executed steps |
| **New Agent** | 3% | Build DatabaseAgent | Agent registry usage logs |
| **Critical Bug Fix** | $50-500 bounty | Fix security vuln | One-time payout |
| **Performance** | 2% | Reduce exec time 30%+ | Before/after benchmarks |
| **Documentation** | Community credits | Tutorials, docs | GitHub attribution |

**Payout Mechanics:**
```python
# src/raas/billing.py
monthly_revenue = $10,000
contributor_pool = monthly_revenue * 0.20  # $2,000

# Distribute based on usage
for contributor, usage_pct in usage_tracker.items():
    payout = contributor_pool * usage_pct
    polar.payout(contributor, payout)  # Min $50 threshold
```

### SOP 2: Revenue Share Calculation

**Monthly Process (1st of each month):**

```python
# 1. Calculate total revenue
revenue = polar.get_monthly_revenue()  # e.g., $10,000

# 2. Calculate contributor pool (20%)
pool = revenue * 0.20  # $2,000

# 3. Track usage per contributor
usage = {}
for mission in missions_completed:
    for file in mission.files_used:
        contributor = git.blame(file).author
        usage[contributor] = usage.get(contributor, 0) + mission.credits

# 4. Distribute pool
total_credits = sum(usage.values())
for contributor, credits in usage.items():
    share = credits / total_credits
    payout = pool * share
    if payout >= 50:  # Min threshold
        polar.payout(contributor, payout)
```

### SOP 5: Emergency Response

**When Production Fails:**

```bash
# 1. Check CI/CD status
gh run view  # See what failed

# 2. Rollback if needed
git revert HEAD
git push origin main

# 3. Notify team
telegram.send "🚨 Production rollback: [link]"

# 4. Debug in staging
mekong debug "Check production logs"
```

**Severity Levels:**
| Level | Response Time | Action |
|-------|---------------|--------|
| P0 (Critical) | < 15 min | All-hands, rollback immediately |
| P1 (High) | < 1 hour | On-call engineer fixes |
| P2 (Medium) | < 24 hours | Next business day |
| P3 (Low) | < 1 week | Backlog priority |
