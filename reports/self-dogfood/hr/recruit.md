# First Hire: Community Manager — Job Description
*Mekong CLI v5.0 | Open Source | Hiring Target: Q3 2026 at $3K MRR*

## Role Overview

**Title:** Community Manager (Part-Time, Contract)
**Type:** Part-time contractor, 20 hrs/week
**Compensation:** $1,200–1,800/mo (based on experience)
**Location:** Remote, async-first
**Start:** When MRR hits $3,000 (trigger-based hire)

---

## Why This Role First

Community Manager is the highest-leverage first hire for an open source CLI tool because:

1. **Stars drive installs drive trials drive revenue** — community growth is the top-of-funnel
2. **Support load grows faster than code** — GitHub issues, Discord questions, and PR reviews need human attention
3. **Founder time is the scarcest resource** — every hour spent on Discord is an hour not spent on product

This role directly unlocks the growth loop: better community → more stars → more trials → more revenue → fund engineering hire.

---

## Responsibilities

### Community Operations (40%)
- Monitor and respond to Discord channels daily (< 4hr response time during business hours)
- Triage GitHub Issues: label, reproduce, route to appropriate owner
- Welcome new Discord/GitHub members personally
- Run weekly community office hours (30 min async AMA in Discord)
- Compile weekly "community pulse" summary for founder

### Content & Engagement (35%)
- Write 2–3 Twitter/X threads per week from real usage patterns
- Curate and amplify community showcase posts (`#showcase` channel)
- Maintain `good-first-issues` list — ensure 10+ labeled at all times
- Write monthly "What's new in Mekong" newsletter digest
- Identify power users and invite them to contributor program

### Growth (25%)
- Submit Mekong to developer tool directories, newsletters, and showcases weekly
- Coordinate with founder on Product Hunt, HN, and Reddit launch moments
- Track community metrics (stars, Discord members, issue response time) weekly
- Recruit affiliate partners from community (bloggers, YouTubers)

---

## Requirements

### Must Have
- Comfortable using a terminal and reading GitHub diffs
- Experience managing a developer community (Discord, GitHub, Slack)
- Strong async written communication in English
- Self-directed — can work without daily check-ins
- Genuine interest in AI tools and developer automation

### Nice to Have
- Python basics (can run `pip install`, read error tracebacks)
- Experience with open source projects as contributor or maintainer
- Twitter/X presence in the dev/indie hacker community
- Prior use of Zapier, n8n, or similar automation tools

### Not Required
- Deep programming experience (this is not a coding role)
- Prior startup experience
- Degree

## Interview Process

1. **Application:** Cover letter + link to a developer community you've managed (GitHub org, Discord, forum)
2. **Async task (30 min):** Triage 5 sample GitHub issues — label, write response, identify if it's a bug/feature/question
3. **30-min video call:** With founder — culture fit, async communication style, genuine interest in the product
4. **Offer:** Trial month at $600 (half rate), then full rate if metrics hit
