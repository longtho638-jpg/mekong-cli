# Go-to-Market Strategy — Mekong CLI

**Model:** Product-led growth via OSS + content | **Budget:** $0 cash

---

## GTM Philosophy

Mekong CLI is a developer tool. Developers don't respond to ads — they respond to:
1. GitHub presence (stars, activity, quality README)
2. Content (technical articles, honest build-in-public)
3. Community (Discord, issues, contributor culture)
4. Word of mouth (one builder tells another)

Paid acquisition is not viable at $49 ARPU with $0 CAC budget. Content compounds.

---

## Phase 1: GitHub Presence (April 2026)

**Goal:** Make the repo impossible to ignore.

| Action | Effort | Impact |
|--------|--------|--------|
| Polish README with demo GIF | 4 hrs | High — first impression |
| Add architecture diagram | 2 hrs | High — credibility |
| Write CONTRIBUTING.md | 2 hrs | Medium — signals community intent |
| Add `good first issue` labels | 1 hr | Medium — contributor magnet |
| GitHub Topics: ai, cli, python, business-automation | 30 min | Medium — discoverability |
| Pin repo on profile | 5 min | Low — profile visitors |

**Target:** Repo looks production-ready before any external promotion.
