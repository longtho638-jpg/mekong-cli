## Mục tiêu:
CTO phải phối hợp 3 panes THÔNG MINH thay vì chạy độc lập.

## Yêu cầu implementation:

### 1. Cross-Pane Event System
- Khi P0 (mekong-cli core) hoàn thành task → CTO đọc output → tạo follow-up task cho P1/P2 nếu liên quan
- Ví dụ: P0 upgrade CTO brain code → P1/P2 cần restart để dùng code mới
- Detect completion patterns: "Phase X hoàn thành", "Cooked for Xm", CC CLI idle sau khi busy

### 2. Output Quality Assessment
- CTO phải ĐỌC output của CC CLI (không chỉ state busy/idle)
- Dùng LLM để evaluate: task thành công hay thất bại?
- Nếu fail → auto-retry hoặc escalate
- Nếu success → log kết quả + trigger dependent tasks

### 3. Smart Task Routing
- Phân loại task theo project: mekong-cli → P0, well → P1, algo-trader → P2
- Nếu 1 pane idle quá lâu → CTO tự generate task phù hợp
- Priority queue: CRITICAL > HIGH > MEDIUM > LOW

### 4. Implementation location:
- Tạo file mới: `lib/cross-pane-orchestrator.js`
- Hook vào `auto-cto-pilot.js` sau LLM Vision loop
- Export: `evaluateOutput(paneIdx, output)`, `triggerDependentTasks(paneIdx, result)`, `routeTask(taskContent)`

### 5. Tests:
