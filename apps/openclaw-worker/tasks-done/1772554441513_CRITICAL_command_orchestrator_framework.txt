## BỐI CẢNH CHIẾN LƯỢC (Chairman Decree):
- Ngoài kia đầy repo skills rời rạc (/cook, /debug, /plan) — COMMODITY, không giá trị
- Giá trị SÁT THỦ của mekong-cli = MULTI-AGENT ORCHESTRATION + 100 Vibe SDK packages
- Dev cần: 1 lệnh → spawn N agents → mỗi agent dùng đúng SDK package → kết quả tổng hợp
- Ví dụ: `/mekong launch fintech-app` → agent 1 dùng fintech-hub-sdk, agent 2 dùng vibe-payment, agent 3 dùng vibe-auth → ra app hoàn chỉnh

## YÊU CẦU IMPLEMENTATION:

### 1. Command Registry (`src/commands/registry.py`)
- Scan tất cả packages/ → tự build command map
- Mỗi package có metadata (industry, capabilities, dependencies)
- Command format: `/mekong <verb> <industry>-<type>` 
- Ví dụ: `/mekong launch fnb-pos`, `/mekong audit healthcare-app`, `/mekong bootstrap ecommerce-store`

### 2. Multi-Agent Orchestrator (`src/commands/orchestrator.py`)  
- Nhận 1 command → decompose thành N parallel tasks
- Mỗi task = 1 agent + 1 SDK package context
- Agent coordination: shared state, dependency resolution
- Output aggregation: merge N results thành 1 deliverable

### 3. SDK-Aware Task Planner (`src/commands/sdk_planner.py`)
- Input: user intent (e.g. "build a restaurant ordering app")
- Output: task graph with SDK assignments
  - Task 1: scaffold UI → vibe-ui + vibe-fnb
  - Task 2: payment → vibe-payment + vibe-pos  
  - Task 3: auth → vibe-auth
  - Task 4: deploy → vibe-edge + vibe-observability

### 4. Killer Commands (examples):
```
mekong launch <industry>      # Full app from SDK template + multi-agent build
mekong compose <pkg1> <pkg2>  # Combine N packages into working app
mekong swarm <task>           # Spawn parallel agents for complex task  
mekong audit <project>        # Multi-agent quality scan using all relevant SDKs
mekong evolve <project>       # Self-improving loop: scan → fix → test → repeat
```

### 5. Files to create/modify:
- `src/commands/registry.py` — Package scanner + command mapping
