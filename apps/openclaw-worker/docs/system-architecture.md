# System Architecture

## Overview
OpenClaw Worker (Code Name: **Tom Hum**) is an autonomous task dispatch daemon responsible for orchestrating missions within the Mekong ecosystem.

## Core Components

### 1. Task Dispatcher (`task-watcher.js`)
- Monitors `tasks/` directory for new mission files (FIFO queue).
- Write-ahead log (WAL) for crash recovery in `~/.openclaw/`.
- Routes missions to appropriate sub-projects or executes them locally.
- Manages brain lifecycle via `brain-process-manager.js` facade.

### 2. The Brain (`lib/brain-process-manager.js` facade + 14 sub-modules)

**Facade Pattern:** `brain-process-manager.js` (37 lines) re-exports public API; all logic in sub-modules.

#### Core Brain Modules (Lifecycle Management):
- **`brain-logger.js`** — Centralized logging (leaf module, zero dependencies)
- **`brain-state-machine.js`** — State tracking (IDLE → SPAWNING → RUNNING → CRASHED)
- **`brain-boot-sequence.js`** — Dual-mode boot: direct (`claude -p`) or tmux session
- **`brain-spawn-manager.js`** — Process spawning, stream setup, command generation
- **`brain-mission-runner.js`** — Mission execution coordinator, result collection
- **`brain-respawn-controller.js`** — Rate-limited respawn (max 5/hour)
- **`brain-terminal-app.js`** — VS Code terminal integration

#### Reliability Modules (Fault Tolerance):
- **`circuit-breaker.js`** — State machine (CLOSED → OPEN → HALF_OPEN) with configurable thresholds
- **`brain-heartbeat.js`** — File-based liveness check (age-based staleness detection)
- **`brain-output-hash-stagnation-watchdog.js`** — Detect hung processes via output hash drift
- **`mission-recovery.js`** — Model failover (HTTP 400) and context overflow recovery
- **`brain-system-monitor.js`** — Thermal/RAM monitoring with M1 cooling triggers

#### Observability Modules (Health & Metrics):
- **`brain-health-server.js`** — HTTP health endpoint on port 9090
  - `GET /health` — Returns 200 (OK), 503 (CRITICAL)
  - `GET /metrics` — Prometheus-compatible metrics
  - Optional Bearer token auth
- **`agi-score-calculator.js`** — AGI score: 5 dimensions × 20 pts = 100/100 max
  - Dimensions: Reliability, Autonomy, Learning Capacity, Safety, Throughput
- **`api-rate-gate.js`** — Rate limiting for mission dispatch

#### Mission Management:
- **`mission-dispatcher.js`** — Prompt building, project routing, mission execution
- **`task-queue.js`** — FIFO queue with Dead Letter Queue (DLQ) for failed missions
- **`mission-journal.js`** — Telemetry logging, mission history in `data/mission-history.json`
- **`mission-complexity-classifier.js`** — Classify mission complexity for routing

#### Autonomous Planning (AGI L4+):
- **`auto-cto-pilot.js`** — Self-CTO: generates Binh Phap quality tasks (console_cleanup, type_safety, a11y_audit, etc.)
- **`project-scanner.js`** — Tech debt & build issue scanning (AGI L4 Shi Ji)
- **`learning-engine.js`** — Meta-learning & pattern analysis (AGI L5 Yong Jian)
- **`evolution-engine.js`** — Self-correction, strategy optimization, brain surgery triggers
- **`post-mission-gate.js`** — CI/CD gate (npm run build verification) & auto-commit (AGI L3)

#### Support Modules:
- **`m1-cooling-daemon.js`** — M1 thermal/RAM protection (runs every 90s)
- **`live-mission-viewer.js`** — Real-time colored log viewer for VS Code terminal

### 3. Health & Monitoring

**New Health Endpoint (Port 9090):**
```bash
# Health check
curl http://localhost:9090/health

# Metrics (Prometheus format)
curl http://localhost:9090/metrics
```

**AGI Score Dimensions:**
- Reliability: circuit breaker state + heartbeat freshness
- Autonomy: tasks generated by auto-CTO vs user-submitted
- Learning: reusable skills synthesized from missions
- Safety: ethical guardrails (Binh Phap safety-guard)
- Throughput: missions/hour, success rate, avg duration

### 4. AI DevKit Integration
- **Role**: Tactical Execution Engine.
- **Components**:
  - **CLI**: `ai-devkit` for interactive workflow guidance.
  - **Memory**: `@ai-devkit/memory` MCP server for session context.
- **Workflow ("Hybrid V")**:
  - **Strategy**: ClaudeKit agents (`/plan`, `/scout`) define *what* to do.
  - **Tactics**: AI DevKit commands (`/execute-plan`, `/code-review`) guide *how* to do it step-by-step.

## Data Flow
1. **Mission Ingestion**: `mission_*.txt` placed in `tasks/`.
2. **Dispatch**: `task-watcher.js` parses mission and spawns Brain.
3. **Execution**: Brain uses ClaudeKit/AI DevKit tools to modify code/files.
4. **Verification**: `post-mission-gate.js` runs builds/tests.
5. **Completion**: Result logged to `mission-journal.js` and task moved to `tasks/processed/`.
