# Algo Trader 📈

Modular algorithmic trading bot with RSI+SMA strategy, backtesting, and multi-exchange support via CCXT.

## Quickstart (3 commands)

```bash
npm install
npm run setup         # Interactive wizard — enter API keys, everything auto-configures
npm run quickstart    # Demo backtest + system status + next steps
```

Backtest works immediately without API keys. For live trading/arbitrage, enter your exchange keys during setup.

## Alternative: One-Click Shell Script

```bash
./scripts/one-click-setup-and-start.sh
```

Handles prerequisites check, dependency install, setup wizard, and optional Docker infrastructure.

## Manual Setup

1.  **Install dependencies:**
    ```bash
    npm install
    ```
2.  **Configure environment:**
    ```bash
    cp .env.example .env
    # Edit .env — only API keys required, everything else has smart defaults
    ```
3.  **Run:**
    ```bash
    npm run dev backtest              # No API key needed
    npm run dev arb:agi               # Needs 2+ exchange keys
    ```

**IMPORTANT:** Never commit your `.env` file to version control.

## Architecture Overview

The Algo Trader is designed with a modular architecture to allow for flexible strategy development, robust risk management, and adaptable exchange integration.

```
src/
├── core/             # Core components: BotEngine, OrderManager, RiskManager, StrategyLoader
│   ├── BotEngine.ts          # Orchestrates the trading process
│   ├── OrderManager.ts       # Handles order placement, cancellation, and tracking
│   ├── RiskManager.ts        # Enforces risk parameters (e.g., position sizing, stop-loss)
│   └── StrategyLoader.ts     # Dynamically loads trading strategies
├── strategies/       # Implementations of various trading strategies
│   ├── RsiSmaStrategy.ts     # RSI + SMA crossover strategy
│   ├── RsiCrossoverStrategy.ts # RSI overbought/oversold signals
│   └── ... (other arbitrage strategies)
├── execution/        # Exchange communication layer
│   └── ExchangeClient.ts     # CCXT wrapper for interacting with various exchanges
├── analysis/         # Technical indicator calculations
│   └── indicators.ts         # RSI, SMA, etc.
├── backtest/         # Backtesting framework
│   └── BacktestRunner.ts     # Simulates strategies against historical data
├── data/             # Data providers for backtesting and live trading
│   └── MockDataProvider.ts   # Provides mock historical data for testing
├── reporting/        # Trade reporting and performance analysis
│   ├── ConsoleReporter.ts    # Outputs trade logs to console
│   ├── HtmlReporter.ts       # Generates detailed HTML reports
│   └── PerformanceAnalyzer.ts # Calculates key performance metrics
├── ui/               # User interface components (e.g., CLI dashboard)
│   └── CliDashboard.ts       # Command-line interface for monitoring
├── interfaces/       # TypeScript interfaces for core entities
│   ├── IConfig.ts            # Configuration interface
│   ├── IStrategy.ts          # Strategy interface
│   └── ...
└── utils/            # Utility functions and helpers
    ├── config.ts             # Configuration loading and management
    └── logger.ts             # Centralized logging service
```

## Usage Guide

### Running Backtests

To test a strategy against historical data without risking real capital:

```bash
# Ensure ENABLE_BACKTESTING is set to true in your .env file.
# Configure the strategy parameters (e.g., TRADING_PAIR, TIMEFRAME) in .env or config files.

# Build the project
npm run build

# Execute a backtest (example for RsiSmaStrategy)
npm run backtest --strategy=RsiSmaStrategy --pair=BTC/USDT --timeframe=1h
# Output will be generated by the configured reporters (e.g., console, HTML report)
```

### Running Live Trading

To deploy the bot for live trading on an exchange:

```bash
# Ensure ENABLE_LIVE_TRADING is set to true in your .env file.
# Verify your EXCHANGE_API_KEY and EXCHANGE_SECRET are correctly set in .env.
# Double-check all trading and risk management parameters.

# Build the project
npm run build

# Start live trading (example for RsiSmaStrategy)
npm start --strategy=RsiSmaStrategy --pair=BTC/USDT --timeframe=1h
# The bot will connect to the exchange and begin trading based on the strategy.
# Monitor its activity via the CLI dashboard or configured logging.
```

**IMPORTANT:** Live trading involves real financial risk. Thoroughly test your strategies with backtests and paper trading before deploying to a live account.

### arb:spread (Cross-Exchange Spread Detector)

```bash
# Default: BTC/USDT + ETH/USDT on Binance/OKX/Bybit, auto-execute
npx ts-node src/index.ts arb:spread

# Dry-run (scan only, no trades)
npx ts-node src/index.ts arb:spread --dry-run

# Custom config
npx ts-node src/index.ts arb:spread \
  -p BTC/USDT,ETH/USDT \
  -e binance,okx,bybit \
  -s 500 \
  -t 0.08 \
  --score-threshold 60 \
  --max-loss 50

# Limited polls (stop after 100 cycles)
npx ts-node src/index.ts arb:spread --max-polls 100
```

**Features:**
- Real-time spread dashboard every 5 cycles
- Signal scoring + orderbook validation (auto-exec mode)
- Circuit breaker with max daily loss protection
- `--dry-run` for safe monitoring without execution

### arb:agi (Recommended for Production)

```bash
# Default: full AGI pipeline with regime detection + Kelly sizing
npx ts-node src/index.ts arb:agi

# Custom config
npx ts-node src/index.ts arb:agi \
  -p BTC/USDT,ETH/USDT \
  -e binance,okx,bybit \
  -s 1000 \
  --score-threshold 65 \
  --max-loss 100

# Disable optional AGI features
npx ts-node src/index.ts arb:agi --no-regime --no-kelly --no-self-tune

# Custom regime detection interval (ms)
npx ts-node src/index.ts arb:agi --regime-interval 15000
```

**AGI Features:**
- **Regime Detection** — Hurst exponent + volatility ratio classifies market (trending/mean-reverting/volatile/quiet)
- **Kelly Position Sizing** — Optimal position sizing based on historical win rate and payoff ratio
- **Self-Tuning** — Score/spread thresholds auto-adjust via EMA of profitability
- **Strategy Routing** — Execution parameters adapt per detected regime
