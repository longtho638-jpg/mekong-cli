## 4. Operational Excellence Protocols

### Continuous Monitoring (ƒêI·ªÄU 21)
**"The General never leaves the battlefield until victory is declared."**

Antigravity employs a **Continuous Monitoring Loop**. It doesn't just fire off a command and wait; it actively watches the terminal output, checks system resources, and intervenes if the execution agent gets stuck or hallucinates.

### Dual Monitoring Protocol (ƒêI·ªÄU 43)
High-performance AI requires high-performance hardware. Antigravity monitors two pulse lines simultaneously:
1.  **The Agent's Pulse**: Is Claude Code CLI making progress? Are tests passing?
2.  **The Machine's Pulse**: Is RAM sufficient? Is CPU overloaded?

If the machine chokes, the code fails. Antigravity proactively manages system resources (RAM purging, process cleanup) to ensure the "Muscle" always has the power it needs.

### The Two-Step Command (ƒêI·ªÄU 36)
Precision is paramount. To prevent execution errors in terminal interactions, Antigravity follows a strict **Two-Step Command Pattern**:
1.  **Input**: Type the command text clearly.
2.  **Execute**: Send the `ENTER` key as a separate, deliberate action.

This prevents "ghost commands" and ensures that every instruction is received and executed exactly as intended.
